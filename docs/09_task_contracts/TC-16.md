## Why this contract exists

The build wired the pipeline for **offline certification** with canned Node A/C (TC-15) and built the LLM adapter **components** (TC-06: `NodeA.extract`, `NodeC`, `resolve_provider_call`), but never created a work item to **assemble those components into a live-provider run**. The CLI intentionally refuses non-certification `burhan run` (exit 10, "real-provider study runs land with a later contract"). This is that contract. It is assembly + I/O plumbing over existing parts — **not** new statistics and **not** new governance rules.

## Objective & Scope

Expose a governed live-provider execution path that takes a study document through real Node A extraction → optional researcher glance → live Node C Gate 1 → the existing headless Stage-1A registry → results store, with deterministic-replay archival so NFR-101 holds despite non-deterministic LLM calls.

**In scope:**
1. **Live run entrypoint** — a CLI mode (e.g. `burhan run <study-dir>` without `--certification`, or an explicit `--live`) that routes to a new `live_run` orchestration path. The certification path stays exactly as-is (canned, offline).
2. **Document ingestion** — a DOCX→text step (python-docx, already locked) producing the `study_document` text Node A consumes; the survey-instrument DOCX likewise → data-dictionary text. The raw CSV remains ordinary pipeline data and is **never** sent to any LLM (existing allowlist enforced).
3. **Real provider settings** — load the real `llm.yaml` through the existing adapter boundary (`resolve_provider_call`), with the startup lineage check (A≠C, FR-304) and prompt-hash-into-manifest already built in TC-06.
4. **Node A live extraction + write-back** — Node A extracts `study_config.yaml` from the document text, validates against schema + V1–V7, **persists** it into the study bundle's `config/`, and hashes the persisted bytes into the manifest.
5. **Pause-before-Gate-1** — after extraction and write-back, the run **halts and surfaces the extracted contract** for the optional researcher glance; it **cannot proceed to Gate 1 without explicit researcher confirmation** (a resume token / confirmation input). This pause is mandatory-capable and un-bypassable by design.
6. **Live Node C Gate 1** — on confirmation, real Node C audits the contract against the source document (bounded retries per policy `gates.max_retries`).
7. **Headless Stage-1A** — execute the **existing** production registry with the real nodes and approved governance; no stage internals change.
8. **LLM archival + replay (NFR-101)** — archive every live LLM prompt+response into the sealed run directory; `burhan rerun` **replays the archived responses** instead of re-calling providers, yielding byte-identical artifacts. This is the determinism mechanism for a live run.
9. **Manifest/provenance** — hash the actual consumed config, policy, playbook, prompt templates, `llm.yaml` settings (model IDs, not keys), and the archived LLM responses. Keys never enter the manifest.
10. **Reference comparison** — execute the TC-12 reference-comparison builder against the results store and the §3 protocol worksheet to emit `REFERENCE_COMPARISON.md`.

**Explicitly out of scope:**
- Any change to Stage-1A statistical modules, thresholds, or outputs (adapters/orchestration only; if a change seems needed, STOP and escalate).
- Any change to the certification path (it must remain canned and offline — proven by test).
- Stage-1B real behavior (narrate/package remain their TC-13/14 selves; M6 exercises Stage-1A, and the run may stop after the reference comparison per protocol §2).
- Governed-document edits; new dependencies (python-docx, anthropic, openai already locked).
- Perplexity / Semantic Scholar or any literature-search integration — not part of M6.

## Acceptance Tests (binding)

- **AT-M16-1 (live path is real):** a `live_run` against a fixture study (small DOCX + tiny CSV, a **stubbed-but-non-canned** provider that records calls) drives document→Node A→persisted `study_config.yaml`→pause→Gate 1→Stage-1A→`COMPLETED`; the provider stub is invoked (proving the live path, not the canned echo).
- **AT-M16-2 (certification unchanged):** the certification path remains canned and offline — its existing tests pass unchanged; an assertion proves `certification_run` still injects canned nodes and calls no network client (AT-M15 suite green).
- **AT-M16-3 (pause is un-bypassable):** the run cannot reach Gate 1 without the researcher confirmation token; a test proves that without confirmation the run halts at the glance and performs no Node C call and no Stage-1A work.
- **AT-M16-4 (no raw data to LLMs):** the CSV bytes/paths never reach any LLM adapter — the allowlist rejection is exercised on the live path; only document/dictionary text and the results-derived artifacts are eligible (NFR-401).
- **AT-M16-5 (rerun identity via replay):** `burhan rerun` on a sealed live run replays archived LLM responses (no provider calls — proven by a provider stub that fails if called) and regenerates byte-identical artifacts (NFR-101); a planted archive mismatch is caught.
- **AT-M16-6 (write-back + manifest):** the persisted `study_config.yaml` is hashed into `hashes.study_config`; the archived LLM responses and `llm.yaml` model IDs are hashed into the manifest; keys appear nowhere in manifest or logs.
- **AT-M16-7 (DOCX ingestion):** the DOCX→text step yields Node A's input from the fixture DOCX; a corrupt/unreadable DOCX fails cleanly (typed halt), never silently empty.
- **AT-M16-8 (no Stage-1A drift):** the whole prior unit + golden + benchmark + integration suite stays green — proving no statistical or certification behavior changed.
