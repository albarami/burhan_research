## Why this contract exists

The build spec specified the orchestrator's behavior (M04) and every stage's internals (M05–M14) but never created a work item to **register the stages into the orchestrator's DAG**. Consequently `burhan run` correctly refuses (exit 10) with stages unregistered, and certification C4 (IT-1..IT-4) cannot pass. This contract closes exactly that gap — integration, not new analysis.

## DAG semantics (reviewer Option A — resolves the prior conflict)

The orchestrator enforces **exact coverage of the fixed 13-stage DAG**: S0 INGEST, G1 GATE-1, S2 POWER, S3 PREP, S4 ASSUMPTIONS, S5 MEASUREMENT, S6 STRUCTURAL, S7 EFFECTS, S8 ROBUSTNESS, S9 NARRATE, G2 GATE-2, S10 PACKAGE — plus S1 CONTRACT (Node A). `burhan run` cannot reach `COMPLETED` with any stage unregistered. Therefore TC-15 wires **all thirteen**, but with a principled split of depth:

- **Stage-1A stages (S0–S8, S1, G1)** — wired as thin adapters over their **already-certified** modules (`ingest/crosswalk`, `contract/node_a`, `review/node_c`, `prep/*`, `stats/*`, `verify/*`, governance, playbook/compliance, provenance). No new statistical behavior.
- **Stage-1B stages (S9 NARRATE, G2 GATE-2, S10 PACKAGE)** — wired as **certification pass-through stubs**: deterministic, minimal stages that satisfy the `Stage` protocol, advance the state machine, and emit schema-valid placeholder artifacts, so `burhan run` completes. Their **full behavior is delivered by TC-13 (narrate) and TC-14 (package)**; TC-15 must not implement narration, the number-resolution checker, or the reporting set. The stub boundary is explicit in code and asserted by AT-M15-6.

This lets IT-1..IT-3 exercise the real Stage-1A pipeline end-to-end while keeping Stage-1B logic with its owning contracts.

## Objective & Scope

Wire the fixed 13-stage DAG into the existing orchestrator so the pipeline executes end-to-end under deterministic stubbed LLM nodes, with Stage-1A stages running their certified modules and Stage-1B stages running certification pass-throughs — satisfying IT-1..IT-3.

**In scope:**
- Register all 13 DAG stages into the orchestrator's stage registry via the `Stage` protocol (M04).
- Stage-1A adapters: thin wrappers calling existing module functions; add no statistical behavior.
- Stage-1B pass-through stubs: deterministic, schema-valid, minimal; clearly marked and boundaried.
- Deterministic **stubbed** Node A / Node C fixtures (canned, schema-valid; no provider calls).
- Production stage registry returns the 13 wired stages (replacing the empty `{}`).
- `tests/integration/` IT harness delivering IT-1, IT-2, IT-3.
- A `burhan doctor` marker line printing `BURHAN_CERTIFIED_WORKSTATION` status (P3 closure).
- Update `docs/00_DOC_INDEX.md` to reflect TC-15 as a researcher-authored remediation contract (the index currently reads "TC-01…TC-14").

**Explicitly out of scope:**
- Any change to a Stage-1A module's statistical behavior, thresholds, or outputs — adapters call existing code. If an adapter cannot be built without changing module behavior, STOP and escalate.
- Any **real** Stage-1B behavior — narration, the number-resolution/claim checker, APA tables, SPSS/AMOS pack, governance docs. Those are TC-13/TC-14. TC-15 delivers only pass-through stubs for S9/G2/S10.
- Governed-document edits beyond the DOC_INDEX update named above.
- New dependencies.
